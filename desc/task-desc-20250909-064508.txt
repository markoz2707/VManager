ANALYSIS OVERVIEW:
==================

The Storage endpoint provides comprehensive VHD/VHDX management functionality through a REST API. The implementation consists of multiple layers: controller, service interface, and WMI-based service implementation.

STORAGE ENDPOINT METHODS:
=========================

1. CREATE VHD - POST /api/storage/vhd
   - Purpose: Creates a new virtual hard disk file
   - HTTP Method: POST
   - Request Body: CreateVhdRequest (JSON)
   - Parameters:
     * Path: File path for the new VHD/VHDX
     * MaxInternalSize: Maximum size in bytes
     * Format: "VHD" or "VHDX"
     * Type: "Dynamic" or "Fixed"
   - Response: 200 OK on success
   - Implementation: Uses Msvm_ImageManagementService.CreateVirtualHardDisk

2. ATTACH VHD - PUT /api/storage/vhd/attach
   - Purpose: Attaches a VHD/VHDX file to a virtual machine
   - HTTP Method: PUT
   - Query Parameters:
     * vmName: Name of the target virtual machine
     * vhdPath: Path to the VHD/VHDX file to attach
   - Response: 200 OK on success, 500 with error details on failure
   - Implementation: Uses WMI XML manipulation to bypass read-only property restrictions
   - Features:
     * Automatic controller selection (prefers SCSI over IDE)
     * Automatic address allocation
     * Support for both SCSI and IDE controllers

3. DETACH VHD - PUT /api/storage/vhd/detach
   - Purpose: Detaches a VHD/VHDX file from a virtual machine
   - HTTP Method: PUT
   - Query Parameters:
     * vmName: Name of the source virtual machine
     * vhdPath: Path to the VHD/VHDX file to detach
   - Response: 200 OK on success
   - Implementation: Uses Msvm_VirtualSystemManagementService.RemoveResourceSettings

4. RESIZE VHD - PUT /api/storage/vhd/resize
   - Purpose: Resizes an existing VHD/VHDX file
   - HTTP Method: PUT
   - Request Body: ResizeVhdRequest (JSON)
   - Parameters:
     * Path: Path to the VHD/VHDX file to resize
     * MaxInternalSize: New maximum size in bytes
   - Response: 200 OK on success
   - Implementation: Uses Msvm_ImageManagementService.ResizeVirtualHardDisk

VIRTDISK.DLL USAGE AND PURPOSE:

What is virtdisk.dll:
- Windows system library for Virtual Hard Disk operations
- Part of Windows since Windows 7/Server 2008 R2
- Provides low-level VHD/VHDX manipulation capabilities
- Used by Windows built-in tools like Disk Management, PowerShell

Where virtdisk.dll is used:
1. VHD Creation: CreateVirtualDisk API for direct file creation
2. VHD Opening: OpenVirtualDisk API for accessing existing files
3. VHD Attachment: AttachVirtualDisk API for mounting to host system
4. VHD Detachment: DetachVirtualDisk API for unmounting from host
5. VHD Resizing: ResizeVirtualDisk API for expanding/shrinking files
6. VHD Information: GetVirtualDiskInformation API for metadata queries

Advantages of virtdisk.dll approach:
- No WMI overhead - direct system calls
- Better performance for bulk operations
- More granular control over VHD operations
- Native Windows error codes and handling
- Works independently of Hyper-V services

Limitations of virtdisk.dll approach:
- Attaches VHDs to host system, not to specific VMs
- Requires additional logic for VM-specific operations
- Limited to VHD file operations (no VM management)
- Requires elevated privileges for some operations
