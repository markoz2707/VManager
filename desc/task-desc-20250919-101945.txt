Task Description - Fix Metrics Collection Problem
Started: 2025-09-19 10:19:45
Completed: 2025-09-19 10:58:35

TASK OBJECTIVE:
Fix the problem with metrics collection where metrics should be enabled for all VMs automatically. If metrics are not enabled, enable them at first check. The current log from agent showed:
"Enumerating discrete metrics for VM 'test-vm-fixed-demo'
Retrieved VM path: \\DESKTOP-MAREK\root\virtualization\v2:Msvm_ComputerSystem.CreationClassName="Msvm_ComputerSystem",Name="919408B8-3952-4CC5-951E-D5C9AB86B1D6"
Found 0 aggregate metrics for VM 'test-vm-fixed-demo'
No aggregate metrics found for VM 'test-vm-fixed-demo'. Metrics collection may not be enabled. Consider calling EnableMetricsCollection first."

PROBLEM ANALYSIS:
1. The current MetricsService.EnableMetricsCollection method was using an incorrect approach by manually creating Msvm_MetricDefForME associations
2. The Microsoft sample code shows that the proper way is to use the Msvm_MetricService::ControlMetrics method
3. The EnumerateDiscreteMetricsForVm method was not automatically enabling metrics when none were found
4. The implementation was not following the Microsoft Hyper-V WMI samples properly

SOLUTION IMPLEMENTED:

Phase 1 - Analysis of Microsoft Sample Code:
1. Examined Microsoft Metrics/EnumerateMetrics.cs sample
2. Examined Microsoft Metrics/ControlMetrics.cs sample  
3. Examined Microsoft Metrics/MetricUtilities.cs sample
4. Identified that ControlMetrics method should be used instead of manual association creation

Phase 2 - Fix EnableMetricsCollection Method:
1. Replaced manual Msvm_MetricDefForME association creation with proper ControlMetrics approach
2. Added ControlMetrics helper method based on Microsoft sample:
   - Takes managedElementPath, metricDefinitionPath, operation, and scope parameters
   - Uses Msvm_MetricService::ControlMetrics WMI method
   - Properly validates output using WmiUtilities.ValidateOutput
3. Added GetMetricService helper method based on Microsoft sample:
   - Gets Msvm_MetricService instance from WMI
   - Uses WmiUtilities.GetFirstObjectFromCollection for proper object retrieval
4. Updated EnableMetricsCollection to use ControlMetrics with operation=2 (Enable)
5. Support for enabling all metrics (pass null to ControlMetrics) or specific metrics

Phase 3 - Auto-Enable Metrics in EnumerateDiscreteMetricsForVm:
1. Modified EnumerateDiscreteMetricsForVm to automatically enable metrics when none are found
2. Restructured method to avoid "using variable" compilation error:
   - Split into EnumerateDiscreteMetricsForVm (main method)
   - EnumerateMetricsInternal (handles metric collection logic)
   - ProcessMetricCollection (processes found metrics)
3. Added automatic metrics enabling logic:
   - When no aggregate metrics found, automatically call ControlMetrics to enable all metrics
   - Retry metric enumeration after enabling
   - Provide appropriate response messages for different scenarios
4. Enhanced error handling and logging throughout the process
