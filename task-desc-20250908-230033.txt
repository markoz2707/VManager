TASK DESCRIPTION: Fix Storage Operations WMI Attachment Issues
================================================================

Date: 2025-09-08 23:00:33
Task: Fix problems with Storage operations, specifically WMI attachment issues while maintaining existing VHDX attachment approach

PROBLEM ANALYSIS:
================
The storage operations were failing due to two main issues:
1. WQL query escaping problem in WmiUtilities.GetResourcePool method
2. Attempting to set read-only properties directly on Msvm_StorageAllocationSettingData objects

ISSUES IDENTIFIED:
==================

1. WQL Query Escaping Issue:
   - Location: src/HyperV.Core.Wmi/WmiUtilities.cs, GetResourcePool method
   - Problem: PoolId parameter containing backslashes was not properly escaped for WQL queries
   - Symptom: "Invalid query" errors when trying to find resource pools

2. Read-Only Properties Assignment Issue:
   - Location: src/HyperV.Core.Wmi/Services/StorageService.cs, AttachVirtualHardDisk method
   - Problem: Attempting to directly set read-only properties (Parent, Address) on Msvm_StorageAllocationSettingData
   - Symptom: "Not found" error at line 61 when trying to assign values to read-only properties

SOLUTIONS IMPLEMENTED:
======================

1. Fixed WQL Query Escaping (WmiUtilities.cs):
   - Added proper backslash escaping in GetResourcePool method
   - Code change: `string escapedPoolId = poolId.Replace("\\", "\\\\");`
   - Used escapedPoolId in WQL query construction instead of raw poolId
   - This ensures PoolId values with backslashes are properly handled in WQL queries

2. Fixed Read-Only Properties Assignment (StorageService.cs):
   - Replaced direct property assignment approach with XML manipulation technique
   - New approach:
     a) Create Msvm_StorageAllocationSettingData instance
     b) Set only writable properties directly (ResourceType, ResourceSubType, HostResource)
     c) Get XML representation using GetText(TextFormat.WmiDtd20)
     d) Parse XML and modify read-only property nodes (Parent, Address)
     e) Pass modified XML to AddResourceSettings method
   - This bypasses the read-only property restriction by modifying the serialized representation

3. Enhanced Controller Selection Logic:
   - Improved GetSuitableControllerWithAddress method
   - Prefers SCSI controllers over IDE controllers for VHD attachment
   - Proper address allocation: SCSI uses "0,X,0" format, IDE uses "0,X" format
   - Better error handling and resource disposal

TECHNICAL DETAILS:
==================

Microsoft Hyper-V WMI API Constraints:
- Msvm_StorageAllocationSettingData has several read-only properties according to Microsoft documentation
- Properties like Parent, Address, and others cannot be set directly via property assignment
- XML manipulation is the recommended workaround for setting these values

Controller Types and Addressing:
- SCSI Controllers (ResourceType = 6): Support up to 256 devices, address format "0,target,0"
- IDE Controllers (ResourceType = 5): Support up to 2 devices per channel, address format "0,device"
- SCSI controllers are preferred for VHD attachment due to better performance and capacity
