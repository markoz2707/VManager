## 🔧 IMPLEMENTATION DETAILS

### Container Configuration Differences:
**HCS Containers (Lightweight)**:
- Uses container images (e.g., nanoserver, servercore)
- Process isolation or Hyper-V isolation
- Direct container runtime integration
- Faster startup, lower overhead

**WMI Containers (Hyper-V Isolated)**:
- Uses WMI DefineSystem with container type
- Full Hyper-V isolation
- Traditional VM-like security boundary
- Higher overhead, stronger isolation

### Service Architecture:
```
ContainersController
├── HCS ContainerService (lightweight containers)
│   ├── HcsCreateComputeSystem
│   ├── HcsStartComputeSystem  
│   ├── HcsPauseComputeSystem
│   └── HcsTerminateComputeSystem
└── WMI ContainerService (Hyper-V isolated)
    ├── DefineSystem (VirtualSystemType: Container)
    ├── RequestStateChange
    └── Job monitoring
```
