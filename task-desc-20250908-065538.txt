ARCHITECTURE NOTES:
===================

The implementation follows the established dual backend pattern:
- HCS backend for container-like VMs (limited snapshot support)
- WMI backend for traditional Hyper-V VMs (full feature support)
- Intelligent fallback from HCS to WMI when operations are not supported
- Consistent JSON API responses regardless of backend used

The snapshot association breakthrough using VirtualSystemIdentifier was critical for proper functionality, as the SystemName property is empty for snapshots and association queries only work for currently applied snapshots.
